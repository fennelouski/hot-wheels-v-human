# Core/RaceCore/ — physics, race rules, tuning (Phase 2 — the fun lives here)

Files to create
- `RaceTuning.swift` — **exists since Phase 1** with the constants Models/TrackKit need (gravityScale, chassis/tire tables, lane offsets, waypoint spacing, loop min speed, elevation level height, max pieces, lives). Phase 2 adds: drive force per chassis, boost impulse (start 1.5 N·s), meter charge time (8 s), destruction thresholds (fall −1 m, stuck <0.05 m/s ×3 s, flipped ×3 s), respawn delay (2 s), snapshot rate (10 Hz). One flat enum, obsessively commented.
- `CarFactory.swift` — `CarDesign` → car `Entity`: USDZ chassis + 4 wheel models attached, box collision (`ShapeResource.generateBox` from visual bounds ×0.9), `PhysicsBodyComponent(massProperties:material:mode:.dynamic)`, paint applied by walking model materials → tinted `PhysicallyBasedMaterial`. Adds `CarComponent` (playerID, lane) + `LaneFollowComponent` (spline ref, progress index). The riding driver is painted by `DriverPainter` from `design.driver`.
- `DriverPainter.swift` (C4) — `DriverProfile` → the Quaternius human: generates the rig's 32×32 stripe-palette texture (rows = skin/eyes/hair/shirt/pants, ranges in `DriverPalette.StripeRows`; bald = hair stripe painted skin-tone), nearest-sampled, no mipmaps, cached per color combo. Then hands off to `DriverDressUp`. Used by racing, the reaction cam, and the editor preview.
- `DriverDressUp.swift` (C5) — procedural hats/glasses/hair-volume props (`MeshResource.generate*`, no assets), attached at the head; `HeadPinSystem` re-pins the wardrobe to the posed `Head` joint every frame (fixed bind-pose offset as fallback — RaceTuning.driverHeadHeightRatio).
- `DriveSystem.swift` — RealityKit `System`. Per car per frame: find target waypoint ahead (lookahead ~0.3 m), apply drive force along spline tangent, PD-steer torque toward tangent, soft "magnet" force toward spline when |offset| < 0.15 m (keeps slot-car feel while letting big physics violations win — flying off IS the game). Applies queued boost impulses.
- `RaceRulesSystem.swift` — checkpoint crossing (piece boundaries), lap/finish detection, destruction checks per RaceTuning, debris spawn (`debris-*` models, random impulses, 3 s despawn), life decrement, respawn at last checkpoint after delay, win/lose determination. Emits `RaceEvent`s via delegate closure (RaceCoordinator forwards to transport).
- `RaceCoordinator.swift` — host-side state machine `lobby → collectingDesigns → buildingTrack → countdown → racing → paused/finished → results`. Owns transport events in, snapshots out @10 Hz, boost validation (server-side meter), AI opponent driver.
- `AIBoostPolicy.swift` — easy/medium/hard boost timing per PRD §6.4 (reads upcoming piece types from the spline/piece map).
- `PhysicsEvents.swift` — collision-event subscription → crash SFX triggers + reaction-cam event feed.

   **State of the physics (sim drills: `--solo-arena [--car 0/1] [--tires ...]`, `[race]` lines in console AND `Documents/drill-log.txt` via `simctl get_app_container … data`):** the demo race now completes cleanly — both demo cars finish with sane top speeds, no catapults, no eternal races. The fixes that got here, in causal order: cars collide with SOLVED geometry (bed slabs per piece + a box chain riding the loop's own centerline) instead of the models' exact meshes; cars drop-spawn (`spawnLift`) and sit visible-and-dynamic-but-inert through the countdown (`RaceEventBus.raceActive` gates DriveSystem/rules) so the Simulator's multi-second warmup stalls land on parked cars; a hitch-rollback in RaceSession restores car poses after any stalled frame (`hitchRollbackThreshold`); the spline follower re-acquires its lane after small skips (window kept short so the loop can't be skipped) and stops driving past the lane's end; respawns go two pieces back; `stuckSpeed` 0.3 ends dawdling; `finishCatchRadius` 0.7 stops fast finishers slipping through the line between frames. The monster truck has been traced climbing the full ring. **Loops are guaranteed** by the loop motor (`RaceTuning.loopCarrySpeed`/`loopSpeedCap`/`loopMotorAccel`): on a loop piece the slot grips the car like the booster wheels in a real Hot Wheels set — cars slower than the carry speed get pushed, cars hotter than the cap get braked into the 3–4 m/s band the ring rides cleanly. No car is ever stranded by a loop; boosted fly-offs elsewhere remain the game.
